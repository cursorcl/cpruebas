Resumen

El código en esta carpeta contiene lo necesario para generar, entrenar, salvar y cargar una red neural que reconozca imágenes.

IMPORTANTE:
Notar que por alguna razón desconocida (al interior de la librería de redes neurales), es preciso aumentar el tamaño del stack para salvar y cargar redes desde archivo, lo que se puede lograr con el argumento -Xss<value> a la VM.
El valor -Xss64m parece ser suficiente.

Creación de la red

La aplicación cl.triton.card.TrainerPlus levanta un panel donde se puede crear, entrenar, probar y salvar redes neurales.

Parece ser que la dimensión predefinida (64 x 24) de la entrada es suficiente para el objeto de reconocer (al menos) 10 tipos de imágenes distintas del interés actual.
Este valor se podría poder cambiar en los spinner antes de crear la red.

La secuencia que funciona es:
  1. Definir el tamaño de la entrada (que representa la dimensión a la cual se van a escalar las imágenes) y crear la red (botón Create network). La creación puede demorar un tanto, paciencia por favor.
  2. Cargar las imágenes a entrenar (botón Load Image, la imagen se muestra en el panel central) y por cada una indicar el índice que se desea recuperar (spin Training output) y (sólo) entonces agregar esa imagen al set de entrenamiento (botón Add training set).
    Cada vez que se agrega una muestra, crece el valor al costado derecho del botón, indicando el número de muestras actualmente en el set de entrenamiento.
  3. Una vez que se han agregado todas las muestras (puede agregarse más de una para cada salida deseada) y sólo entonces se entrena la red (botón Train). El entrenamiento puede demorar un resto, paciencia por favor.
  4. Una vez entrenada la red, conviene probarla con algunas imágenes, tanto pertenecientes al set de entrenamiento como otras diferentes, para chequear los resultados. Para ello cargar la imagen con Load Image y pruébela con Recognize.
    El resultado se muestra en el arreglo de valores que aparece bajo la imagen, ordenados desde el índice cero (0) en adelante. El rango de valores debe estar entre cero (no hay calce) y uno (calce perfecto).
    Se supone que el que tiene el máximo lejos sobre el resto es el índice resultado.
  5. Salve la red en un archivo (botón Save network).
    Aunque al comienzo es entretenido, después es una lata estar repitiendo la cuestión.

Este entrenador asume que se parte desde cero. Ignoro si es posible (y, por ahora, me da lo mismo) si se pueden agregar nuevos datos y re-entrenar una red ya entrenada y cosas por el estilo.
Total, no demora ni tanto y se hace una sola vez (o pocas veces, en el peor caso).

Uso en producción

Una función de reconocimiento está representada en la interfaz cl.cursor.card.Recognizer, que debería parecerse a: 
public interface Recognizer
{
  /**
   * Recognize an image.

   * If the recognition process resolve a solution into the given tolerance,
   * then that answer index is returned, else -1.
   * 
   * @param image
   *          The image to be recognized.
   * @param tolerance
   *          The tolerance to admit a solution. A value in percentage that
   *          should be done between the best solution versus the others.
   * @return The index of the best solution if into tolerance, -1 otherwise.
   */
  int recognize(Image image, double tolerance);

  /**
   * Test the image and return the recognition result.
   * 
   * @param image
   *          The image to be recognized.
   * @return The recognition process output.
   */
  double[] match(Image image);
}

Supongo que se explica por sí sola. Por si acaso, hay que darle una imagen al método recognize, que devuelve el índice reconocido, et voilá.
El parámetro tolerance es cuánto se está dispuesto a transar en la resolución del reconocimiento. Una tolerancia de 0.5 expresa que el mejor valor obtenido debe superar al resto al menos por el 50% de su valor (normalmente es mucho mejor que eso).

Para generar una instancia de la interfaz anterior, basta usar el archivo salvado con la red entrenada en algo así como: 
  Recognizer recognizer = RecognizerFactory.create(file-with-the-trained-net);

La aplicación cl.cursor.card.testbed.TestTrainedRecognizer prueba exactamente eso. Es casi lo más simple que se puede hacer para probar el concepto.
